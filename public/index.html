Quick start

Sign up, create a live key, upload a sliced file, and submit a print job — all over HTTP.

curl · bash 
# 1) Create a merchant account (returns a setup token in full-auto mode)
curl -X POST https://bambu-print-farm-automation.vercel.app/api/public/merchants/signup \
  -H 'Content-Type: application/json' \
  -d '{"company_name":"Acme Prints","contact_email":"ops@acme.com","password":"a-strong-password"}'

# 2) Exchange the setup token for a live API key
curl -X POST https://bambu-print-farm-automation.vercel.app/api/public/api-keys \
  -H 'Content-Type: application/json' \
  -d '{"setup_token":"pkx_setup_...","name":"storefront"}'

# 3) Submit a print job with your live key
curl -X POST https://bambu-print-farm-automation.vercel.app/api/public/print-jobs \
  -H 'Authorization: Bearer pkx_live_...' \
  -H 'Content-Type: application/json' \
  -d '{"file_id":"...","printer_preference":{"model":"X1C"}}'

# 4) Poll status
curl https://bambu-print-farm-automation.vercel.app/api/public/print-jobs/status?job_id=...

Platform

Why PrintKinetix

Built specifically for Bambu Lab fleets — the cloud orchestrates, the edge executes.

Printers stay private

The Windows node talks to printers over your LAN (FTPS + MQTT). Nothing on the public internet touches a printer directly.

Automated farm loops

Cool-release ejection and N-loop runs are baked into the sliced file, so finished parts clear automatically and the next run starts.

AMS-aware routing

Live filament inventory feeds the router, so jobs land on a printer with the right material — and idle capacity — without manual mapping.

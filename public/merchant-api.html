PrintKinetix Merchant API v1

Submit merchant print files, route them to the best available Bambu printer, and track usage through the live PrintKinetix cloud control plane.

Merchant onboarding | OpenAPI JSON | Farm admin | Windows node guide

Onboarding

  1. Use the public merchant onboarding form or call POST /api/public/merchants/signup with company and contact details.
  2. Signup is approval-required by default. Pending merchants receive no live credentials.
  3. An admin approves the merchant with POST /api/cloud/merchants.
  4. The admin issues a one-time pkx_setup_... token during approval or with POST /api/cloud/merchant-setup-token.
  5. The merchant exchanges that setup token at POST /api/public/api-keys for a live pkx_live_... API key, or an admin issues a live key with POST /api/cloud/merchant-api-keys.

Full-auto merchant signup exists as an admin option, but full_auto_merchant_mode defaults to disabled.

The Vercel farm admin at /cloud also manages smart material queueing, filament/AMS inventory, auto-eject policy, failure-detection hooks, and integrations for alerts, ecommerce, shipping, and remote access through GET/PATCH /api/cloud/farm-automation.

Public farm capabilities are available at GET /api/public/farm/capabilities with accepted file types, routing strategies, max build volume, enabled automation features, and safe filament availability.

Merchants can check public material and color availability with GET /api/public/farm/filaments. The response is aggregate-only and does not expose spool IDs, printer IDs, rack locations, lot codes, or reserved jobs.

Authentication

Merchant live API key

Authorization: Bearer pkx_live_...

One-time setup token exchange

X-Merchant-Setup-Token: pkx_setup_...

Admin cloud token

Authorization: Bearer <CLOUD_ADMIN_TOKEN>

Idempotent print submission

Idempotency-Key: order-1001-print-v1

Submit A Print Job

POST /api/public/print-jobs. Use fastest_fulfillment, batch_by_material, least_printer_wear, or ship_cutoff in options.routing_strategy.

{
  "name": "Order 1001",
  "file": {
    "name": "part.gcode.3mf",
    "content_type": "application/octet-stream",
    "base64": "..."
  },
  "requirements": {
    "dimensions_mm": { "x": 100, "y": 100, "z": 80 },
    "materials": ["PLA"],
    "colors": ["#ffffff"]
  },
  "options": {
    "merchant_order_id": "1001"
  }
}
File typeBehavior
.gcode, .3mf, .gcode.3mfStored privately, routed with fastest_fulfillment, and queued for the selected local node when capacity matches.
.stl, .obj, .step, .stpStored and tracked as needs_slicing until slicer automation is attached.

Quote, Preflight, Webhooks

POST /api/public/quotes estimates material grams, queue time, print time, price, and lead time without creating a job.

POST /api/public/print-jobs/preflight validates file type, build volume, routing capacity, and review warnings before upload.

GET/POST /api/public/webhooks manages signed merchant webhooks. Deliveries include X-PrintKinetix-Event, X-PrintKinetix-Timestamp, and X-PrintKinetix-Signature.

GET /api/public/integrations lists supported Shopify, WooCommerce, Etsy, ShipStation, Slack, Zapier, Make, and webhook integrations.

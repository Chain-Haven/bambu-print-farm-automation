Windows Local Node

Run one Windows computer on the same local network as the Bambu printers. It keeps an outbound secure connection to Vercel and executes printer work over LAN.

Merchant onboarding | Merchant API docs | Cloud console

Network Model

  • The Windows node only needs HTTPS outbound access to the Vercel app.
  • Do not open inbound firewall ports from the public internet to the Windows node.
  • The node stores LOCAL_NODE_TOKEN, not SUPABASE_SERVICE_ROLE_KEY, CLOUD_ADMIN_TOKEN, or token pepper values.
  • The node talks to printers through LAN MQTT and FTPS after the printer is configured locally.
  • Final command results are kept in cloud-result-outbox.json and replayed if Vercel or Supabase is temporarily unavailable.
  • Heartbeats report non-secret IPv4 interface inventory so operators can confirm multi-network printer coverage.

Setup

  1. Open /cloud and enter the bootstrap admin token.
  2. Use Download Windows App to create or reuse the organization, provision a node through /api/cloud/nodes, and download the prefilled, self-contained app through /api/cloud/node-package.
  3. Copy the one-time LOCAL_NODE_TOKEN only if you need to repair .env.
  4. Extract the app to a stable folder such as C:\PrintKinetix.
  5. Double-click Start Farm Node.bat. There is no npm install and no manual Node install — every dependency is compiled into farm-node.cjs, and the launcher uses a bundled Node runtime, a Node already on the PC, or downloads a portable Node automatically the first time.
  6. Open http://localhost:3000 on that Windows computer and add each Bambu printer with its LAN IP, serial, and access code.
  7. Return to /cloud, verify the node heartbeat is online, then queue Discover LAN Printers and Sync Printer Inventory.

Printer Sync Commands

cloud.printers.discover returns Bambu SSDP discoveries visible from the Windows machine. cloud.printers.sync returns saved local printers, live worker status, AMS tray counts, and filament snapshots when available.

{
  "command_type": "cloud.printers.sync",
  "payload": {
    "scan_cidrs": ["192.168.1.0/24"],
    "include_saved_printers": true,
    "sync_ams": true,
    "sync_filament": true
  }
}

Print Command

The merchant API queues cloud.print.ready when a ready file can be routed. The Windows node downloads the signed private artifact, wraps raw .gcode into .gcode.3mf when needed, uploads it to the selected printer over LAN FTPS, and starts the print over Bambu MQTT.

{
  "command_type": "cloud.print.ready",
  "payload": {
    "local_printer_id": "printer-1",
    "download_url": "https://...",
    "original_name": "part.gcode.3mf"
  }
}
